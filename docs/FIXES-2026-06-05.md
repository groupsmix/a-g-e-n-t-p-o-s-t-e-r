# Critical fixes — 2026-06-05

End-to-end fix pass for the bug report. After verifying each claim against
the actual source, three of the six "bugs" turned out to be issues in dead
code (a Supabase-based dashboard scaffold that never matched the real
Cloudflare D1 + Cloudflare Pages architecture). Those got removed, not
patched. The genuinely live bugs in NEXUS were fixed.

## Architecture (ground truth)

| Surface          | Stack                                                       | Location                                |
| ---------------- | ----------------------------------------------------------- | --------------------------------------- |
| Dashboard UI     | Next.js 14 on Cloudflare Pages (`@cloudflare/next-on-pages`) | `apps/nexus/apps/web` (`@nexus/web`)    |
| API              | Cloudflare Worker + D1 + KV + R2                            | `apps/nexus/apps/nexus-api`             |
| AI worker        | Cloudflare Worker + Workers AI                              | `apps/nexus/apps/nexus-ai`              |
| Database         | Cloudflare D1 (SQLite-at-edge)                              | `apps/nexus/apps/nexus-api/migrations/` |

**There is no Supabase. There is no Vercel. There is one user (the owner).**

The `apps/dashboard/` directory from earlier scaffolding work was a
Supabase-based prototype that was never connected to the real architecture.
It is removed in this PR.

## What this PR does

### 1. Remove `apps/dashboard/` and its plumbing

Deleted:

- `apps/dashboard/` (entire directory)
- CI steps in `.github/workflows/ci.yml` that typechecked + built
  `@repo/dashboard` with placeholder Supabase env vars
- The `apps/dashboard/` row in `README.md`

The dashboard is **`apps/nexus/apps/web`**, deploys to **Cloudflare Pages**,
and is documented in `apps/nexus/DEPLOY.md`.

### 2. NEXUS web — visible API misconfig warning

- `apps/nexus/apps/web/src/lib/api.ts` now exports `API_BASE_IS_FALLBACK`
  and `isApiMisconfigured()`. A loud `console.error` fires on module load
  when the page runs on a non-localhost origin with the localhost fallback.
- `apps/nexus/apps/web/src/components/shared/ApiMisconfigBanner.tsx`
  (new): amber banner that appears at the top of every NEXUS page when the
  misconfig is detected. Copy points at the right fix (`pages:ship` or
  Cloudflare Pages env vars).
- `apps/nexus/apps/web/src/components/shell/AppShell.tsx`: wires the banner
  in.

This is intentionally non-breaking. Local dev (`pnpm --filter @nexus/web
dev` on `localhost`) still works exactly as before, because
`isApiMisconfigured()` only fires when the host is not localhost. The
banner is the safety net for the case where someone ships a build without
running `pages:build` (which already injects the correct URL).

### 3. NEXUS web — stub pages for the 3 truly missing routes

The sidebar in `apps/nexus/apps/web/src/components/shell/Sidebar.tsx`
links to `/content`, `/affiliate-marketing`, and `/ecommerce-retail` — but
no `page.tsx` existed at those paths, so clicking landed on a Next.js 404.

Added:

- `apps/nexus/apps/web/src/app/content/page.tsx` — hub that links to the
  live content surfaces (blog, email, competitors)
- `apps/nexus/apps/web/src/app/affiliate-marketing/page.tsx` — stub +
  "for maintainers" note pointing at the missing
  `apps/nexus/apps/nexus-api/src/routes/affiliate.ts`
- `apps/nexus/apps/web/src/app/ecommerce-retail/page.tsx` — stub +
  "for maintainers" note pointing at the missing
  `apps/nexus/apps/nexus-api/src/routes/ecommerce.ts`

The other 5 routes the bug report listed (`/signals`, `/events`,
`/ventures`, `/pipeline`, `/portfolio`) are **not** in the default sidebar
so no user can click them by accident; they have backend routes already
and can get UI later when needed.

### 4. Root `.env.example` cleanup

- Removed Supabase, Vercel, and the dashboard-specific GitHub vars
  (they were only used by the deleted dashboard).
- Added Cloudflare auth vars (`CLOUDFLARE_EMAIL`, `CLOUDFLARE_API_KEY`)
  matching what `apps/nexus/DEPLOY.md` actually uses.
- Documented `NEXT_PUBLIC_API_URL` correctly (build-time, defaulted by
  `pages:build`).

## What this PR does NOT do

- Touch the live Cloudflare Pages site. The fixes ship on next deploy via
  `pnpm --filter @nexus/web pages:ship`.
- Add backend routes for the missing affiliate / ecommerce domains. That's
  a separate feature, not a bug fix.
- Touch the AGENT_TASKS doc. It still contains the historical Supabase
  scaffolding instructions; those are now stale but harmless.
